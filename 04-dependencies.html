Dépendances

Gestion des dépendances

En Python, les dépendances sont gérées à l'aide du gestionnaire de paquets pip.

Pour installer un paquet, vous pouvez utiliser la commande suivante : 

pip install nom_du_paquet

Environnement virtuel

Il est recommandé d'utiliser un environnement virtuel pour isoler les dépendances de votre projet.

Un environnement virtuel permet d'isoler les paquets installés pour un projet spécifique, évitant ainsi les conflits entre différentes versions de paquets.

Il est également possible de spécifier la version de Python à utiliser dans l'environnement virtuel.

Environnement virtuel

Pour créer un environnement virtuel, vous pouvez utiliser la commande suivante : 

python -m venv .venv

Cette commande crée un dossier nommé .venv dans le répertoire courant, contenant l'environnement virtuel.

Pour activer l'environnement virtuel, utilisez la commande suivante : 

source .venv/bin/activate
.venv est le nom de l'environnement virtuel (dossier créé).

Environnement virtuel avec VSCode

Si vous utilisez Visual Studio Code, vous pouvez configurer l'environnement virtuel en ouvrant le panneau de commande (Ctrl + Shift + P) et en sélectionnant "Python: Create Environment".

Installation des dépendances

Pour installer les dépendances d'un projet, vous pouvez utiliser un fichier requirements.txt.

Pour créer ce fichier, vous pouvez utiliser la commande suivante : 

pip freeze > requirements.txt

Pour installer les dépendances à partir de ce fichier, utilisez la commande suivante : 

pip install -r requirements.txt

Cela installera toutes les dépendances listées dans le fichier requirements.txt.
